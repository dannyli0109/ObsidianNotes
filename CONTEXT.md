# DaN Vault

一个游戏开发学习 vault：学习游戏开发过程中的教程、概念、Bug 和代码片段。其他内容不属于这里。

## 术语表

**Daily Note（每日笔记）**：
唯一的记录入口——一次学习会话的一次性草稿。有长期价值的内容会被提炼出去；每日笔记只保留指向已提炼内容的链接。
_避免使用_：日记（Journal）、收件箱（Inbox）

**Promotion（提炼）**：
把有长期价值的内容从每日笔记转移到 Typed Note 中，在每次学习会话的最后 10 分钟完成。

**Typed Note（类型笔记）**：
frontmatter 中带有 `type:` 字段的永久笔记，归档在对应类型的文件夹里。类型笔记才是资产，每日笔记不是。

**Bug Note（Bug 笔记）**：
记录一个问题的笔记，放在 `05 Bugs & Fixes`。唯一带有 `status:` 字段的笔记类型（`open` 或 `fixed`）。未解决的问题可以以 `open` 状态提炼。

**Snippet（代码片段）**：
以后会*再次复制粘贴*的代码，放在 `04 Code Snippets`。保持小巧、可直接复制。

**Concept（概念）**：
以后会想*重新理解*的东西，放在 `03 Learning`（平铺）。如果一篇笔记既是片段又是概念，那它是带代码块的 Concept。

**Tutorial Note（教程笔记）**：
跟学教程时一集/一章的记录，放在 `03 Learning`（平铺）。

**Idea（点子）**：
每个游戏或机制点子一篇笔记，放在 `06 Ideas`。没有模板；一个标题加两句话就够了。

**Project（项目）**：
三个练习代码库之一。`project:` frontmatter 的规范值就是文件夹名原文：
- `Cpp Engine`
- `Godot Practice`
- `Playdate Practice`

**Project README（项目索引）**：
一个项目的索引：顶部是手写的"当前焦点"部分，下方是 Dataview 自动生成的该项目类型笔记列表。
